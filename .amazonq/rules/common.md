# Common best Practices

## Documentation

- Maintain up to date technical documentation in README.md file.
- Keep the documentation simple:
  - How to deploy the infrastructure
  - Global architecture (include an excalidraw scheme)

## Linting and testing

- Use pre-commit to lint the code
- Run "pre-commit run -a" before commiting/pushing to git remote
- Use common hooks from https://github.com/pre-commit/pre-commit-hooks
